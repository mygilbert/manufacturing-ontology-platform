# Manufacturing Ontology Platform - 프로젝트 개요

## 1. 프로젝트 배경

### 문제 정의
반도체/디스플레이 제조 현장에는 FDC, SPC, MES 등 다양한 시스템이 존재하지만, 각 시스템이 **사일로(Silo)** 형태로 운영되어 다음과 같은 문제가 발생합니다:

- 데이터 간 연결성 부재로 근본 원인 분석 어려움
- 시스템별 데이터 형식이 달라 통합 분석 불가
- 엔지니어 경험 지식이 암묵지로 남아 공유 어려움
- 이상 발생 시 수동으로 여러 시스템 조회 필요

### 해결 방향
**팔란티어 파운드리(Palantir Foundry)** 의 온톨로지 개념을 적용하여:

1. 모든 데이터를 **그래프 기반 온톨로지**로 통합
2. **명시적 관계**(FK 등)와 **암묵적 관계**(통계 분석)를 모두 표현
3. **도메인 전문가 지식**을 구조화하여 AI Agent에 반영
4. 실시간 이상 감지 및 근본 원인 자동 추적

## 3. 핵심 개념

### 온톨로지(Ontology)란?
데이터의 **개념(Concept)** 과 **관계(Relationship)** 를 정의하는 지식 표현 체계입니다.

```
예시:
- Equipment "ETCH-001"은 Process "ETCH_STEP_1"을 수행한다
- Process "ETCH_STEP_1"은 Recipe "RECIPE_A"를 사용한다
- Alarm "HIGH_TEMP"가 Lot "LOT-2024001"에 영향을 준다
```

### 명시적 관계 vs 암묵적 관계

| 구분 | 명시적 관계 | 암묵적 관계 |
|------|------------|------------|
| 정의 | 이미 알려진 관계 | 데이터 분석으로 발견 |
| 예시 | FK, 공정 순서 | 파라미터 상관관계 |
| 확보 방법 | 스키마 정의 | 통계/ML 분석 |
| 신뢰도 | 100% | 분석 결과에 따름 |

## 4. 기대 효과

### 정량적 효과
- 이상 발생 시 근본원인 분석 시간: 수 시간 → 수 분
- 데이터 조회 시스템 수: 5-10개 → 1개 통합 뷰
- 엔지니어 암묵지 문서화율: 0% → 80%+

### 정성적 효과
- 신규 엔지니어 교육 시간 단축
- 데이터 기반 의사결정 문화 정착
- 시스템 간 데이터 연결성 가시화
- AI Agent 활용 기반 마련
